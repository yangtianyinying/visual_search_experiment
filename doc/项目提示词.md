# 项目提示词

## 实验概要

实验名称：视觉搜索实验

假设/效应：随着刺激数量增加，被试反应时上升，准确率可能下降；目标存在条件下反应更快且更准确。

被试时长：约 15-25 分钟（含练习与区块间休息）

总体结构：3 个 block，每个 block 16 个 trial，总计 48 个 trial

## 刺激与素材（选择性填充，不填我就默认设定）

视觉刺激：形状刺激，包含四种图形组合：红色正立 T、红色倒立 T、蓝色正立 T、蓝色倒立 T；每 trial 呈现刺激数量为 5/10/15/20（四水平）。屏幕上有 5x5 的网格，每个网格都是一个 T 出现的位置

## 实验流程

试次结构：
1) fixation：中心注视点 500 ms；
2) image：刺激呈现，直到按键或达到 2 s；
3) key_resp：记录键盘反应；
4) feedback：按键或 2 s 超时后反馈（`CORRECT!` / `WRONG!` / `TOO SLOW!`）；
5) trial 间间隔（ITI）：用于降低顺序与预测效应。
时间控制：基于时钟控制；fixation 固定 500 ms，刺激显示最多 2000 ms，反馈紧随反应或超时触发。
响应收集：目标为红色正立 T；出现时要求尽快按键，未出现时不按键；记录按键、RT、正确性、超时状态。
随机化/平衡：每个 block 内采用平衡随机；各条件组合出现次数固定，trial 顺序随机；可加约束避免连续 trial 完全同条件。
条件与因子：被试内设计；因子 1=刺激数量（5/10/15/20），因子 2=目标红色正立 T 是否出现（有/无）；每个组合每 block 重复 2 次，共 4*2*2=16 trial/block。
自适应程序（可选）：不使用 Staircase 或 QUEST。
练习与反馈：正式实验前可设置练习试次，规则与正式实验一致，反馈文案同正式实验。
区块/休息：共 3 个 block；每个 block 16 trial；block 间休息 1-2 分钟。

## 数据记录

输出目录：`data/participant01` 这样类推
文件命名：`{expName}_{participant}_{date}.csv`
数据列：`trialN`、`block`、`stimCount`、`targetPresent`、`respKey`、`corr`、`RT`、`timeout`、`trialStart`、`stimOnset`、`respTime`、`feedbackOnset`、`trialEnd`
时间戳：记录实验开始时间、trial 开始/结束时间、刺激呈现 onset、响应时间点与反馈时间点。

## 被试与会话

信息采集：弹窗采集：participant, age, gender 三项
分组/平衡：当前为单组被试内设计；条件在各 block 内平衡随机。
本地化：中文

## 误差与退出
