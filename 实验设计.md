# 线上实验设计

## 技术

静态网页，发布到 GitHub Pages，结束后将数据下载到用户本地。

前端：HTML、CSS、jsPsych（无后端；GitHub Pages 静态托管）。

无数据库；数据仅存于被试本机（浏览器下载 CSV，UTF-8 BOM）。

项目入口：`index.html`（根目录）。本地调试请在项目根目录启动静态服务器后访问，避免 `file://` 跨域问题。

## 范式：经典视觉搜索

比较 **特征搜索**（颜色 pop-out）与 **联合搜索**（颜色 × 朝向须同时匹配）的表现差异。

- **目标**：蓝色竖条。
- **特征搜索**：干扰物均为红色竖条；目标在颜色维度上单独突出。
- **联合搜索**：干扰物为红色竖条与蓝色横条混合；需同时分辨颜色与朝向。

反应：**J** = 目标存在，**F** = 目标不存在。刺激呈现至按键或 3000 ms 超时。

## 流程与试次

- 被试信息表单（编号、年龄、性别）→ 说明 → **练习** 8 试次 → 正式实验。
- 正式实验：**3 个 block** × **每 block 16 试次** = 48 试次。
- 每 block 内条件：2（搜索类型）× 3（集合大小 8 / 12 / 16）× 2（目标有 / 无）= 12 种组合，各至少 1 次；另随机重复 4 个试次以凑满 16。

## 记录变量（CSV 中 `phase=stimulus` 行）

试次编号、被试编号、人口学变量、`searchType`、`setSize`、`targetPresent`、按键、正误、RT、是否超时、时间戳等。

## 部署（GitHub Pages）

将本仓库推送到 GitHub，在仓库 Settings → Pages 中选择分支与根目录（或 `/docs`），保存后使用站点 URL 访问即可。
