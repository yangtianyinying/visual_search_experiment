# 线上实验设计

## 技术

静态网页，发布到 GitHub Pages，结束后将数据下载到用户本地。

前端：HTML、CSS、jsPsych（无后端；GitHub Pages 静态托管）。

无数据库；数据仅存于被试本机（浏览器下载 CSV，UTF-8 BOM）。

项目入口：`index.html`（根目录）。本地调试请在项目根目录启动静态服务器后访问，避免 `file://` 跨域问题。

## 范式与流程（与 `doc/实验结构.md` 一致）

刺激为四种 **T**：红色正立、红色倒立、蓝色正立、蓝色倒立。刺激呈现在 **5×5 固定网格的交叉点**上（**不画网格线**）；未放置 T 的交叉点**不显示任何内容**。

- **目标**：红色正立 T。
- **反应（go/no-go）**：仅当出现红色正立 T 时按**空格**；否则不按键。
- **刺激时长**：至按键或 2000 ms。
- **集合大小**：5、10、15、20 个 T（从 25 个网格点中随机选取对应数量呈现）。

## 区块

- 练习：8 试次。
- 正式：**3 block × 16 试次** = 48 试次。每 block：4（集合大小）× 2（目标有/无）× 2 次重复 = 16，block 内随机顺序。

## 记录变量（CSV 中 `phase=stimulus` 行）

试次编号、被试编号、人口学变量、`setSize`、`targetPresent`（红色正立 T 是否出现）、按键、正误、RT、是否超时、时间戳等。

## 部署（GitHub Pages）

将本仓库推送到 GitHub，在仓库 Settings → Pages 中选择分支与根目录（或 `/docs`），保存后使用站点 URL 访问即可。
